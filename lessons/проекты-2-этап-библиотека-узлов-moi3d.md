# Библиотека стандартных узлов в Moi3D — заглушки, подвесы, крепления

Каждый новый светильник на 60-80% состоит из типовых конструктивных элементов: торцевых заглушек, кронштейнов подвеса, кабельных вводов. Моделировать их заново каждый раз — потеря времени и источник ошибок. Организованная библиотека стандартных узлов в Moi3D сокращает время моделирования нового изделия в 2-3 раза и гарантирует единообразие конструкции.

***

## Зачем нужна библиотека узлов

Инженер-конструктор работает над несколькими проектами одновременно. Без библиотеки типовых элементов каждый светильник моделируется с нуля, даже если 70% геометрии уже существует в предыдущих проектах. Это приводит к трём проблемам.

* **Потеря времени.** Торцевая заглушка под профиль 35x35 мм моделируется за 15-20 минут. За год на 50 проектах это 15-17 часов только на заглушки.

* **Разнобой в конструкции.** Без единого источника один инженер делает фаску 0.5 мм, другой — 1 мм. Детали перестают быть взаимозаменяемыми.

* **Ошибки при передаче на производство.** Каждая заново смоделированная деталь требует проверки. Библиотечный элемент уже проверен и обкатан.

Библиотека решает все три проблемы: элемент создаётся один раз, проверяется, сохраняется в общую папку и вставляется в любой проект за секунды.

***

Ссылка на библиотеку `O:\Проектный отдел\Библиотеки\Библиотека 3D-моделей.3dm`

## Моделирование библиотечного узла: пошаговый подход

Создание элемента для библиотеки отличается от обычного моделирования тремя требованиями: точность размеров, чистота геометрии и удобство вставки в другой проект.

### Подготовка модели

1. **Определите базовую точку.** Каждый библиотечный элемент привязывается к началу координат (0, 0, 0) своей посадочной поверхностью. Заглушка — внутренней плоскостью паза. Кронштейн — плоскостью контакта с корпусом.
2. **Ориентируйте деталь стандартно.** Ось Z — вверх, ось X — вдоль длинной стороны, ось Y — поперёк. Единообразная ориентация избавляет от вращения при вставке.
3. **Назначьте имя объекту.** В Moi3D выделите объект, откройте Properties и задайте осмысленное имя: `Заглушка_35x35_глухая`. Это имя сохраняется в файле и помогает при импорте.

### Проверка качества геометрии

Перед сохранением в библиотеку проверьте модель:

* **Closed Solid** — объект должен быть замкнутым твёрдым телом. Откройте Properties: статус "Closed Solid" означает, что модель пригодна для производства.

* **Отсутствие дублей** — удалите скрытые поверхности и вспомогательные кривые. Команда Edit > Select All, затем проверка количества объектов.

* **Корректные размеры** — используйте команду Measure для контроля ключевых размеров: толщина стенки, диаметр отверстий, межосевые расстояния.

> Важно: библиотечный элемент с ошибкой геометрии тиражирует ошибку на все проекты. Проверяйте один раз — тщательно.

## Организация файловой структуры

Библиотека работает только при строгой организации файлов. Хаотичный единый файл  с десятками файлов без системы именования бесполезна.

### Структура слоев

```
Library/
  Заглушки/
    ЗАГ-35x35-глухая.3dm
    ЗАГ-35x35-кабель.3dm
    ЗАГ-50x50-глухая.3dm
    ЗАГ-50x50-PG7.3dm
  Подвесы/
    ПОД-накладная-60x60.3dm
    ПОД-U-кронштейн-40мм.3dm
    ПОД-трос-1.0мм.3dm
  Крепления/
    КРП-пластина-80x80.3dm
    КРП-клипса-трек-3ф.3dm
  Поворотные/
    ПОВ-шарнир-1ос-M5.3dm
    ПОВ-кардан-M6.3dm
  Кабельные-вводы/
    КВВ-PG7.3dm
    КВВ-PG9.3dm
    КВВ-втулка-6мм.3dm
```

### Правила именования файлов

Каждое имя файла состоит из трёх частей:

* **Префикс** — трёхбуквенный код группы (ЗАГ, ПОД, КРП, ПОВ, КВВ)

* **Тип** — краткое описание конструкции

* **Ключевой размер** — габарит или диаметр, определяющий совместимость

Разделитель — дефис. Без пробелов, без спецсимволов. Формат файла — `.3dm` (родной формат Moi3D и Rhino).

| Префикс | Группа          | Примеры                              |
| ------- | --------------- | ------------------------------------ |
| ЗАГ     | Заглушки        | ЗАГ-35x35-глухая, ЗАГ-50x50-PG7      |
| ПОД     | Подвесы         | ПОД-накладная-80x80, ПОД-трос-1.5мм  |
| КРП     | Крепления       | КРП-пластина-100x60, КРП-клипса-трек |
| ПОВ     | Поворотные узлы | ПОВ-шарнир-1ос-M5, ПОВ-кардан-M6     |
| КВВ     | Кабельные вводы | КВВ-PG7, КВВ-PG9, КВВ-втулка-8мм     |

## Вставка библиотечного элемента в проект

Moi3D не имеет встроенного менеджера библиотек, но поддерживает быстрый импорт через команду Open и Copy/Paste между файлами.

### Способ 1: импорт через File > Open

1. Откройте рабочий файл проекта.
2. Через File > Open откройте файл библиотечного элемента в отдельном окне Moi3D.
3. Выделите все объекты (Ctrl+A), скопируйте (Ctrl+C).
4. Переключитесь на окно проекта, вставьте (Ctrl+V).
5. Переместите элемент в нужную позицию командой Transform > Move.

### Способ 2: импорт через File > Import

1. В рабочем файле выберите File > Import.
2. Укажите путь к `.3dm`-файлу из библиотеки.
3. Объект появится в начале координат рабочего файла.
4. Используйте Move и Rotate для позиционирования.

### Способ 3: Копирование-вставка напрямую из RHINO

1. Откройте библиотеку `O:\Проектный отдел\Библиотеки\Библиотека 3D-моделей.3dm`
2. Перед тем как копировать и вставлять в MOI3D настройте бесшовный перенос деталей из RHINO в MOI3D следуя уроку `Настройка копирования геометрии из Rhino 8 в MoI3D`

### Позиционирование после вставки

Если базовая точка библиотечного элемента задана правильно (на посадочной поверхности в начале координат), вставка сводится к одной операции Move — от точки (0, 0, 0) к точке посадки на корпусе светильника.

> Важно: после вставки убедитесь, что элемент не пересекается с корпусом и не оставляет зазоров. Используйте сечения (Section) для визуальной проверки посадки.

## Поддержание и расширение библиотеки

Библиотека — живой инструмент. Она растёт вместе с количеством проектов и типов светильников.

### Когда добавлять новый элемент

Элемент заслуживает места в библиотеке, если выполняется хотя бы одно условие:

* Деталь уже использовалась в двух и более проектах

* Деталь стандартизирована (размеры привязаны к типоразмеру профиля или стандарту крепежа)

* Деталь будет передаваться на серийное производство

### Версионирование

При доработке библиотечного элемента не перезаписывайте старый файл. Добавьте суффикс версии: `ЗАГ-35x35-глухая-v2.3dm`. Старая версия остаётся доступной для проектов, где она уже используется.

### Общий доступ

Храните библиотеку в сетевой папке или облачном хранилище (Яндекс Диск, Google Drive) с синхронизацией. Каждый инженер работает с одной и той же версией библиотеки.
