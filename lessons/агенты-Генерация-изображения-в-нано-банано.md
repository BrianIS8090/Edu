**Nano Banana** — так называются встроенные в Gemini возможности для создания изображений. Gemini может генерировать и обрабатывать изображения на основе текста, изображений или их комбинации. Это позволяет создавать, редактировать и дорабатывать визуальные материалы с беспрецедентным уровнем контроля.

Nano Banana — это три разные модели, доступные в Gemini API:

* **Nano Banana 2**: модель [Gemini 3.1 Flash Image Preview](https://ai.google.dev/gemini-api/docs/models/gemini-3.1-flash-image-preview) (**`gemini-3.1-flash-image-preview`**). Эта модель является высокоэффективной альтернативой Gemini 3 Pro Image, оптимизированной для работы с большими объемами данных.

* **Nano Banana Pro**: модель [Gemini 3 Pro Image Preview](https://ai.google.dev/gemini-api/docs/models/gemini-3-pro-image-preview) (**`gemini-3-pro-image-preview`**). Эта модель предназначена для профессиональной обработки изображений. Она использует расширенные возможности логического вывода ("мышления") для выполнения сложных инструкций и высококачественной обработки текста.

* **Nano Banana**: модель [Gemini 2.5 Flash Image](https://ai.google.dev/gemini-api/docs/models/gemini-2.5-flash-image) (**`gemini-2.5-flash-image`**). Эта модель отличается высокой скоростью и эффективностью и оптимизирована для выполнения задач с большим объемом данных и низкой задержкой.

Все сгенерированные изображения содержат [водяной знак SynthID](https://ai.google.dev/responsible/docs/safeguards/synthid).

## Генерация изображений (преобразование текста в изображение)

<br />

```
from google import genai
from google.genai import types
from PIL import Image

client = genai.Client()

prompt = ("Create a picture of a nano banana dish in a fancy restaurant with a Gemini theme")
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[prompt],
)

for part in response.parts:
    if part.text is not None:
        print(part.text)
    elif part.inline_data is not None:
        image = part.as_image()
        image.save("generated_image.png")

```

## Редактирование изображений (преобразование текста и изображений в изображения)

**Напоминание**: убедитесь, что у вас есть необходимые права на загружаемые изображения. Не создавайте контент, нарушающий права других лиц, в том числе видео или изображения, которые вводят в заблуждение, оскорбляют или причиняют вред. Использование вами этого сервиса на основе генеративного искусственного интеллекта регулируется нашей [Политикой в отношении запрещенного использования](https://policies.google.com/terms/generative-ai/use-policy).

Предоставьте изображение и с помощью текстовых подсказок добавьте, удалите или измените элементы, измените стиль или цветопередачу.

В следующем примере показана загрузка **`base64`** закодированных изображений. Информацию о нескольких изображениях, больших объемах данных и поддерживаемых типах MIME можно найти на странице [Распознавание изображений](https://ai.google.dev/gemini-api/docs/image-understanding).

<br />

```
from google import genai
from google.genai import types
from PIL import Image

client = genai.Client()

prompt = (
    "Create a picture of my cat eating a nano-banana in a "
    "fancy restaurant under the Gemini constellation",
)

image = Image.open("/path/to/cat_image.png")

response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[prompt, image],
)

for part in response.parts:
    if part.text is not None:
        print(part.text)
    elif part.inline_data is not None:
        image = part.as_image()
        image.save("generated_image.png")

```

### Многооборотное редактирование изображений

Продолжайте генерировать и редактировать изображения в режиме диалога. Для доработки изображений рекомендуется использовать чат или многоэтапный диалог. В следующем примере показано, как создать инфографику о фотосинтезе.

<br />

```
from google import genai
from google.genai import types

client = genai.Client()

chat = client.chats.create(
    model="gemini-3.1-flash-image-preview",
    config=types.GenerateContentConfig(
        response_modalities=['TEXT', 'IMAGE'],
        tools=[{"google_search": {}}]
    )
)

message = "Create a vibrant infographic that explains photosynthesis as if it were a recipe for a plant's favorite food. Show the \"ingredients\" (sunlight, water, CO2) and the \"finished dish\" (sugar/energy). The style should be like a page from a colorful kids' cookbook, suitable for a 4th grader."

response = chat.send_message(message)

for part in response.parts:
    if part.text is not None:
        print(part.text)
    elif image:= part.as_image():
        image.save("photosynthesis.png")

```

## Новинка с моделями Gemini 3 Image

Gemini 3 предлагает современные модели для создания и редактирования изображений. Gemini 3.1 Flash Image оптимизирована для быстрой работы с большими объемами данных, а Gemini 3 Pro Image — для профессионального создания изображений. Эти модели, разработанные для решения самых сложных задач с помощью продвинутых алгоритмов, отлично справляются со сложными многоэтапными задачами по созданию и изменению изображений.

* **Вывод в высоком разрешении**: встроенная функция создания изображений в разрешении 1K, 2K и 4K.

  * **Gemini 3.1 Flash Image** добавляет разрешение 512 (0,5K).

* **Расширенный рендеринг текста**: возможность генерировать разборчивый стилизованный текст для инфографики, меню, диаграмм и маркетинговых материалов.

* **Использование Google Поиска в качестве основы**: модель может использовать Google Поиск как инструмент для проверки фактов и создания изображений на основе данных в реальном времени (например, текущих карт погоды, биржевых графиков, информации о недавних событиях).

  * **Gemini 3.1 Flash Image** добавляет интеграцию Grounding с поиском изображений в Google наряду с веб-поиском.

* **Режим «мышления»**: модель использует процесс «мышления» для обработки сложных запросов. Она генерирует промежуточные «мыслительные образы» (видимые в бэкенде, но не оплачиваемые), чтобы доработать композицию перед созданием окончательного высококачественного изображения.

* **До 14 эталонных изображений**: теперь вы можете использовать до 14 эталонных изображений для создания финального изображения.

* **Новые соотношения сторон**: в Gemini 3.1 Flash Image Preview добавлены [соотношения сторон](https://ai.google.dev/gemini-api/docs/image-generation#aspect_ratios_and_image_size) 1:4, 4:1, 1:8 и 8:1.

### Используйте до 14 эталонных изображений

Модели изображений Gemini 3 позволяют комбинировать до 14 эталонных изображений. Эти 14 изображений могут включать в себя следующее:

| **Предварительный просмотр изображения вспышки Gemini 3.1**                            | **Предварительный просмотр изображения Gemini 3 Pro**                                 |
| :------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------ |
| До 10 изображений объектов с высокой детализацией для включения в итоговое изображение | До 6 изображений объектов с высокой детализацией для включения в итоговое изображение |
| До 4 изображений персонажей для сохранения единообразия                                | До 5 изображений персонажей для сохранения единообразия                               |

<br />

```
from google import genai
from google.genai import types
from PIL import Image

prompt = "An office group photo of these people, they are making funny faces."
aspect_ratio = "5:4" # "1:1","1:4","1:8","2:3","3:2","3:4","4:1","4:3","4:5","5:4","8:1","9:16","16:9","21:9"
resolution = "2K" # "512", "1K", "2K", "4K"

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[
        prompt,
        Image.open('person1.png'),
        Image.open('person2.png'),
        Image.open('person3.png'),
        Image.open('person4.png'),
        Image.open('person5.png'),
    ],
    config=types.GenerateContentConfig(
        response_modalities=['TEXT', 'IMAGE'],
        image_config=types.ImageConfig(
            aspect_ratio=aspect_ratio,
            image_size=resolution
        ),
    )
)

for part in response.parts:
    if part.text is not None:
        print(part.text)
    elif image:= part.as_image():
        image.save("office.png")

```

### Наземная разведка с помощью поиска Google

Используйте [инструмент поиска Google](https://ai.google.dev/gemini-api/docs/google-search) для создания изображений на основе информации в режиме реального времени, например прогнозов погоды, биржевых графиков или новостей о недавних событиях.

Обратите внимание, что при использовании Grounding с Google Поиском и генерацией изображений результаты поиска по изображениям не передаются в модель генерации и исключаются из ответа (см. [Grounding с Google Поиском по изображениям](https://ai.google.dev/gemini-api/docs/image-generation#image-search))

<br />

```
from google import genai
prompt = "Visualize the current weather forecast for the next 5 days in San Francisco as a clean, modern weather chart. Add a visual on what I should wear each day"
aspect_ratio = "16:9" # "1:1","1:4","1:8","2:3","3:2","3:4","4:1","4:3","4:5","5:4","8:1","9:16","16:9","21:9"

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=prompt,
    config=types.GenerateContentConfig(
        response_modalities=['Text', 'Image'],
        image_config=types.ImageConfig(
            aspect_ratio=aspect_ratio,
        ),
        tools=[{"google_search": {}}]
    )
)

for part in response.parts:
    if part.text is not None:
        print(part.text)
    elif image:= part.as_image():
        image.save("weather.png")

```

Ответ включает **`groundingMetadata`** со следующими обязательными полями:

* **`searchEntryPoint`**: Содержит HTML и CSS для отображения необходимых результатов поиска.

### Приземление с помощью Google Поиска по изображениям (3.1 Flash)

**Примечание:** Эта функция доступна только для модели Gemini 3.1 Flash Image.

Функция Grounding с поиском изображений в Google позволяет моделям использовать веб-изображения, найденные с помощью поиска в Google, в качестве визуального контекста для генерации изображений. Поиск изображений — это новый тип поиска в рамках существующего инструмента Grounding с поиском в Google, который работает наряду со стандартным [веб-поиском](https://ai.google.dev/gemini-api/docs/image-generation#use-with-grounding).

Чтобы включить поиск изображений, настройте **`googleSearch`** инструмент в вашем запросе API и укажите **`imageSearch`** внутри **`searchTypes`** объекта. Поиск изображений можно использовать независимо или вместе с веб-поиском.

Обратите внимание, что функцию Grounding в Google Поиске по изображениям нельзя использовать для поиска людей.

<br />

```
from google import genai
prompt = "A detailed painting of a Timareta butterfly resting on a flower"

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=prompt,
    config=types.GenerateContentConfig(
        response_modalities=["IMAGE"],
        tools=[
            types.Tool(google_search=types.GoogleSearch(
                search_types=types.SearchTypes(
                    web_search=types.WebSearch(),
                    image_search=types.ImageSearch()
                )
            ))
        ]
    )
)

# Display grounding sources if available
if response.candidates and response.candidates[0].grounding_metadata and response.candidates[0].grounding_metadata.search_entry_point:
    display(HTML(response.candidates[0].grounding_metadata.search_entry_point.rendered_content))

```

**Требования к отображению**

При использовании функции поиска изображений в Grounding с помощью Google Поиска необходимо соблюдать следующие условия:

* **Указание источника**: вы должны предоставить ссылку на веб-страницу, содержащую исходное изображение («страницу с исходным изображением», а не сам файл изображения), таким образом, чтобы пользователь понял, что это ссылка.

* **Прямая навигация**: если вы также решили отобразить исходные изображения, вы должны предоставить прямой путь к исходной странице с исходными изображениями, по которому можно перейти одним кликом. Любая другая реализация, которая задерживает или ограничивает доступ конечного пользователя к исходной веб-странице, в том числе с помощью нескольких кликов или промежуточного просмотра изображений, не допускается.

**Ответ**

Для обоснованных ответов, основанных на поиске изображений, API предоставляет четкую атрибуцию и метаданные, позволяющие связать результаты с проверенными источниками. К ключевым полям объекта **`groundingMetadata`** относятся:

* **`imageSearchQueries`**: конкретные запросы, используемые моделью для визуального контекста (поиск по изображениям).

* **`groundingChunks`**: содержит исходную информацию для полученных результатов. Для источников изображений они будут возвращены в виде URL-адресов перенаправления с использованием нового типа фрагментов изображения. Этот фрагмент включает в себя:

  * **`uri`**: URL веб-страницы для указания авторства (целевой страницы).

  * **`image_uri`**: URL прямого изображения.

* **`groundingSupports`**: Предоставляет конкретные сопоставления, которые связывают сгенерированный контент с соответствующим источником цитирования в фрагментах.

* **`searchEntryPoint`**: Включает в себя чип «Поиск Google», содержащий совместимые HTML- и CSS-элементы для отображения поисковых подсказок.

### Создавайте изображения с разрешением до 4K

По умолчанию модели изображений Gemini 3 генерируют изображения с разрешением 1000 пикселей на дюйм, но также могут создавать изображения с разрешением 2000, 4000 и 512 (0,5K) (только для Gemini 3.1 Flash Image) пикселей на дюйм. Чтобы создать изображения с более высоким разрешением, укажите **`image_size`** в **`generation_config`**.

Необходимо использовать заглавную букву «K» (например, 1K, 2K, 4K). В значении **`512`** суффикс «K» не используется. Параметры в нижнем регистре (например, 1k) будут отклонены.

<br />

```
from google import genai
from google.genai import types

prompt = "Da Vinci style anatomical sketch of a dissected Monarch butterfly. Detailed drawings of the head, wings, and legs on textured parchment with notes in English."
aspect_ratio = "1:1" # "1:1","1:4","1:8","2:3","3:2","3:4","4:1","4:3","4:5","5:4","8:1","9:16","16:9","21:9"
resolution = "1K" # "512", "1K", "2K", "4K"

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=prompt,
    config=types.GenerateContentConfig(
        response_modalities=['TEXT', 'IMAGE'],
        image_config=types.ImageConfig(
            aspect_ratio=aspect_ratio,
            image_size=resolution
        ),
    )
)

for part in response.parts:
    if part.text is not None:
        print(part.text)
    elif image:= part.as_image():
        image.save("butterfly.png")

```

### Мыслительный процесс

Имитационные модели Gemini 3 — это мыслящие модели, которые используют процесс логического мышления ("Thinking") для обработки сложных запросов. Эта функция включена по умолчанию, и ее нельзя отключить в API. Подробнее о процессе логического мышления можно узнать из руководства [Gemini Thinking](https://ai.google.dev/gemini-api/docs/thinking).

Модель генерирует до двух промежуточных изображений для проверки композиции и логики. Последнее изображение в Thinking также является финальным.

Вы можете проверить ход мыслей, которые привели к созданию финального изображения.

<br />

```
for part in response.parts:
    if part.thought:
        if part.text:
            print(part.text)
        elif image:= part.as_image():
            image.show()

```

#### **Контролирующие уровни мышления**

С помощью Gemini 3.1 Flash Image вы можете контролировать объем мышления, используемый моделью, чтобы сбалансировать качество и задержку. Значение по умолчанию **`thinkingLevel`** равно **`minimal`**, а поддерживаемые уровни - **`minimal`** и **`high`**. Установка **`thinkingLevel`** на **`minimal`** обеспечивает ответы с наименьшей задержкой. Обратите внимание, что минимальное мышление не означает, что модель вообще не использует мышление.

Вы можете добавить логическое значение **`includeThoughts`**, чтобы указать, будут ли сгенерированные моделью мысли отображаться в ответе или останутся скрытыми.

<br />

```
from google import genai

response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents="A futuristic city built inside a giant glass bottle floating in space",
    config=types.GenerateContentConfig(
        response_modalities=["IMAGE"],
        thinking_config=types.ThinkingConfig(
            thinking_level="High",
            include_thoughts=True
        ),
    )
)

for part in response.parts:
    if part.thought: # Skip outputting thoughts
      continue
    if part.text:
      display(Markdown(part.text))
    elif image:= part.as_image():
      image.show()

```

Обратите внимание, что плата за токены для размышлений взимается независимо от того, установлено ли значение **`includeThoughts`** на **`true`** или **`false`**, поскольку [процесс размышления](https://ai.google.dev/gemini-api/docs/image-generation#thinking-process) всегда происходит по умолчанию, независимо от того, наблюдаете вы за ним или нет.

#### **Подписи Мыслей**

Сигнатуры мыслей — это зашифрованные представления внутреннего мыслительного процесса модели, которые используются для сохранения контекста рассуждений при многоэтапном взаимодействии. Все ответы содержат поле **`thought_signature`** . Как правило, если вы получаете сигнатуру мысли в ответе модели, вы должны передать ее в том же виде, в котором она была получена, при отправке истории диалога на следующем этапе. Если сигнатуры мыслей не передаются, ответ может быть некорректным. Подробнее о сигнатурах мыслей можно прочитать в документации [по сигнатурам мыслей](https://ai.google.dev/gemini-api/docs/thought-signatures).

**Примечание:** Если вы используете официальные [Google Gen AI SDK](https://ai.google.dev/gemini-api/docs/libraries) и функцию чата (или добавляете полный объект ответа модели непосредственно в историю), **сигнатуры мыслей обрабатываются автоматически**. Вам не нужно извлекать их вручную, управлять ими или вносить изменения в код.

Вот как работают мысленные сигнатуры:

* Все **`inline_data`** части с изображением **`mimetype`**, которые являются частью ответа, должны иметь подпись.

* Если в начале (до изображения) сразу после мыслей есть несколько текстовых частей, первая текстовая часть также должна иметь подпись.

* Если **`inline_data`** части с изображением **`mimetype`** являются частью мыслей, подписи у них не будет.

## Другие режимы создания изображений

Gemini поддерживает другие режимы взаимодействия с изображениями, основанные на структуре и контексте запроса, в том числе:

* **Преобразование текста в изображение (изображения) и текст (чередование):** выводит изображения с соответствующим текстом.

  * Пример запроса: «Сгенерируйте иллюстрированный рецепт паэльи».

* **Преобразование изображений и текста в изображения и текст (чередование)**: использует входные изображения и текст для создания новых связанных изображений и текста.

  * Пример запроса: (с изображением меблированной комнаты) «Какие еще диваны подойдут для моего интерьера? Можете обновить изображение?»

## Создание изображений в пакетном режиме

Ознакомьтесь с [документацией по созданию изображений с помощью Batch API](https://ai.google.dev/gemini-api/docs/batch-api#image-generation) и [руководством](https://colab.research.google.com/github/google-gemini/cookbook/blob/main/quickstarts/Batch_mode.ipynb) для получения примеров и кода для создания изображений с помощью Batch API.

## Руководство и стратегии по созданию изображений

Освоение создания изображений начинается с одного фундаментального принципа:

> **Опишите сцену, а не просто перечислите ключевые слова.** Главное преимущество модели — глубокое понимание языка. Повествовательный, описательный абзац почти всегда создаст более целостный образ, чем набор разрозненных слов.

### Лучшие практики

Чтобы улучшить свои результаты, внедрите в рабочий процесс эти профессиональные стратегии.

* **Будьте предельно конкретны:** чем больше деталей вы укажете, тем больше у вас будет возможностей для контроля. Вместо «фэнтезийных доспехов» опишите их так: «богато украшенные эльфийские пластинчатые доспехи, инкрустированные сусальным серебром, с высоким воротником и наплечниками в форме соколиных крыльев».

* **Укажите контекст и цель:** объясните *назначение* изображения. Понимание контекста моделью повлияет на конечный результат. Например, запрос «Создайте логотип для элитного минималистичного бренда по уходу за кожей» даст лучшие результаты, чем просто «Создайте логотип».

* **Повторяйте и дорабатывайте:** не ждите идеального результата с первого раза. Используйте диалоговый интерфейс модели, чтобы вносить небольшие изменения. Добавьте подсказки вроде «Отлично, но не могли бы вы сделать освещение чуть более теплым?» или «Оставьте все как есть, но измените выражение лица персонажа, чтобы он выглядел серьезнее».

* **Используйте пошаговые инструкции:** для сложных сцен с большим количеством элементов разбейте запрос на этапы. «Сначала создайте фон — безмятежный туманный лес на рассвете. Затем на переднем плане добавьте покрытый мхом древний каменный алтарь». Наконец, положите на алтарь один светящийся меч.

* **Используйте «семантические отрицательные подсказки»:** вместо того чтобы говорить «без машин», опишите желаемую сцену в позитивном ключе: «пустая, безлюдная улица без признаков движения».

* **Управляйте камерой:** используйте фотографический и кинематографический язык, чтобы контролировать композицию. Используйте такие термины, как **`wide-angle shot`**, **`macro shot`**, **`low-angle perspective`**.

## Ограничения

* For best performance, use the following languages: EN, ar-EG, de-DE, es-MX, fr-FR, hi-IN, id-ID, it-IT, ja-JP, ko-KR, pt-BR, ru-RU, ua-UA, vi-VN, zh-CN.

* Image generation does not support audio or video inputs.

* The model won't always follow the exact number of image outputs that the user explicitly asks for.

* **`gemini-2.5-flash-image`** works best with up to 3 images as input, while **`gemini-3-pro-image-preview`** supports 5 images with high fidelity, and up to 14 images in total. **`gemini-3.1-flash-image-preview`** supports character resemblance of up to 4 characters and the fidelity of up to 10 objects in a single workflow.

* When generating text for an image, Gemini works best if you first generate the text and then ask for an image with the text.

* **`gemini-3.1-flash-image-preview`** Grounding with Google Search does not support using real-world images of people from web search at this time.

* All generated images include a [SynthID watermark](https://ai.google.dev/responsible/docs/safeguards/synthid).

## Дополнительные настройки

При необходимости вы можете настроить способ отображения ответа и соотношение сторон выходного изображения модели в поле **`config`** в **`generate_content`** вызовах.

### Типы выходных данных

По умолчанию модель возвращает текстовые и графические ответы (т. е. **`response_modalities=['Text', 'Image']`**). Вы можете настроить модель так, чтобы она возвращала только изображения без текста, используя **`response_modalities=['Image']`**.

<br />

```
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[prompt],
    config=types.GenerateContentConfig(
        response_modalities=['Image']
    )
)

```

### Соотношения сторон и размер изображения

По умолчанию модель подгоняет размер выходного изображения под размер входного или генерирует квадраты в соотношении 1:1. Вы можете изменить соотношение сторон выходного изображения с помощью поля **`aspect_ratio`** в разделе **`image_config`** в запросе на ответ, как показано здесь:

<br />

```
# For gemini-2.5-flash-image
response = client.models.generate_content(
    model="gemini-2.5-flash-image",
    contents=[prompt],
    config=types.GenerateContentConfig(
        image_config=types.ImageConfig(
            aspect_ratio="16:9",
        )
    )
)

# For gemini-3.1-flash-image-preview and gemini-3-pro-image-preview
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[prompt],
    config=types.GenerateContentConfig(
        image_config=types.ImageConfig(
            aspect_ratio="16:9",
            image_size="2K",
        )
    )
)

```

## Выбор модели

Выберите модель, которая лучше всего подходит для вашего конкретного случая.

* **Gemini 3.1 Flash Image Preview (Nano Banana 2 Preview)** — идеальная модель для создания изображений, сочетающая в себе высокую производительность, интеллектуальные функции, низкую стоимость и баланс между скоростью и задержкой. Более подробную информацию можно найти на страницах [ценообразования](https://ai.google.dev/gemini-api/docs/pricing#gemini-3.1-flash-image-preview) и [возможностей](https://ai.google.dev/gemini-api/docs/models/gemini-3.1-flash-image-preview) модели.

* **Gemini 3 Pro Image Preview (Nano Banana Pro Preview)** предназначен для профессионального создания ассетов и выполнения сложных инструкций. Эта модель использует реальные данные с помощью Google Поиска, а также процесс «мышления» по умолчанию, который дорабатывает композицию перед генерацией, и может создавать изображения с разрешением до 4K. Более подробную информацию можно найти на страницах [цены](https://ai.google.dev/gemini-api/docs/pricing#gemini-3-pro-image-preview) и [возможности](https://ai.google.dev/gemini-api/docs/models/gemini-3-pro-image-preview) модели.

* **Gemini 2.5 Flash Image (Nano Banana)** — это быстрое и эффективное решение. Эта модель оптимизирована для выполнения большого объема задач с низкой задержкой и генерирует изображения с разрешением 1024 пикселей. Более подробную информацию можно найти на страницах [ценообразования](https://ai.google.dev/gemini-api/docs/pricing#gemini-2.5-flash-image) и [возможностей](https://ai.google.dev/gemini-api/docs/models/gemini-2.5-flash-image) модели.

### Когда использовать Imagen

Помимо встроенных в Gemini возможностей создания изображений, вы также можете использовать [Imagen](https://ai.google.dev/gemini-api/docs/imagen), нашу специализированную модель для создания изображений, через Gemini API.

Imagen 4 — идеальная модель для тех, кто только начинает создавать изображения с помощью Imagen. Выбирайте Imagen 4 Ultra для более сложных задач или если вам нужно максимальное качество изображения (обратите внимание, что эта модель может генерировать только одно изображение за раз).
