# Проектирование в DIALux Evo

DIALux Evo — бесплатная профессиональная программа для расчёта освещения. Она позволяет моделировать помещения, размещать светильники, получать расчёт освещённости и формировать отчёты для заказчика и экспертизы. Этот урок — пошаговое руководство от установки до экспорта.

---

## 1. Установка и настройка

### Системные требования

| Параметр | Минимум | Рекомендуемый |
|---|---|---|
| ОС | Windows 10 64-bit | Windows 10/11 64-bit |
| ОЗУ | 4 ГБ | 8–16 ГБ |
| Видеокарта | OpenGL 3.3 | NVIDIA / AMD дискретная |
| Диск | 2 ГБ | SSD, 5+ ГБ |

### Установка

1. Скачать с [dialux.com](https://www.dialux.com/) (бесплатно)
2. Установить, выбрать язык (русский доступен)
3. При первом запуске — создать аккаунт DIALux (для доступа к каталогам)

### Начальная настройка

- **Единицы измерения:** метры (Настройки → Единицы)
- **Нормы:** выбрать EN 12464-1 (европейская) или СП 52.13330 (российская)
- **Расчётная сетка:** по умолчанию — автоматическая (достаточно для большинства задач)

## 2. Импорт фотометрических данных

### Форматы файлов

| Формат | Расширение | Источник |
|---|---|---|
| EULUMDAT | .ldt | Европейские производители |
| IES | .ies | Американские производители |
| ULD | .uld | Формат DIALux |

### Способы добавления светильников

1. **Онлайн-каталог DIALux** — поиск по производителю прямо в программе
2. **Плагины производителей** — скачать с сайта производителя
3. **Импорт файлов** — загрузить .ies/.ldt вручную (Файл → Импорт)

### Где взять файлы для своих светильников

Если вы разрабатываете светильники сами:

1. Заказать фотометрическое измерение в аккредитованной лаборатории
2. Лаборатория выдаст .ies и/или .ldt файл
3. Импортировать в DIALux через «Мои светильники»

## 3. Создание проекта — пошагово

### Шаг 1: Новый проект

**Файл → Новый проект**

Задать:
- Название проекта
- Адрес объекта
- Тип (интерьер / экстерьер / дорога)
- Коэффициент запаса (Maintenance Factor)

### Коэффициент запаса (MF)

Учитывает старение LED, загрязнение светильников и помещения.

| Среда | MF |
|---|---|
| Чистое помещение (офис) | 0,80 |
| Нормальное помещение | 0,67–0,80 |
| Загрязнённое (цех, склад) | 0,50–0,67 |
| Наружное освещение | 0,67–0,70 |

**Важно:** DIALux считает «обслуживаемую» освещённость — то, что останется через несколько лет. Начальная освещённость будет выше на 1/MF.

### Шаг 2: Построение помещения

**Способы:**
1. **Ручной ввод** — задать размеры прямоугольного помещения
2. **Полигон** — нарисовать произвольную форму
3. **Импорт DWG/DXF** — загрузить подоснову из AutoCAD

**Настройки помещения:**
- Высота помещения (от пола до потолка)
- Высота рабочей плоскости (обычно 0,8 м — стол)
- Высота монтажа светильников

### Коэффициенты отражения поверхностей

| Поверхность | Типичное значение |
|---|---|
| Потолок (белый) | 70–80% |
| Стены (светлые) | 50–70% |
| Стены (тёмные) | 20–30% |
| Пол (светлый) | 30–40% |
| Пол (тёмный) | 10–20% |

## 4. Размещение светильников

### Способы размещения

1. **Поле светильников** — равномерная сетка (основной способ для общего освещения)
2. **Линия светильников** — ряд с заданным шагом
3. **Отдельный светильник** — точечное размещение
4. **Линейный (непрерывный ряд)** — для линейных светильников

### Параметры поля светильников

- Количество рядов и светильников в ряду
- Отступ от стен (обычно = 1/2 шага между светильниками)
- Высота подвеса
- Ориентация (вдоль / поперёк помещения)

### Правило размещения

```
Расстояние между светильниками ≤ 1,5 × высота подвеса (от рабочей плоскости)
```

**Пример:** Потолок 3 м, рабочая плоскость 0,8 м → h = 2,2 м → шаг ≤ 3,3 м.

## 5. Расчёт и анализ результатов

### Запуск расчёта

**Кнопка «Рассчитать»** или F5.

### Основные результаты

| Параметр | Обозначение | Единица | Что значит |
|---|---|---|---|
| Средняя освещённость | Ē | лк | Среднее значение на рабочей плоскости |
| Минимальная освещённость | E_min | лк | Наименьшее значение |
| Максимальная освещённость | E_max | лк | Наибольшее значение |
| Равномерность | U₀ = E_min / Ē | — | Должна быть ≥ 0,4–0,6 |
| Мощность освещения | — | Вт/м² | Удельная мощность |
| UGR | — | — | Показатель блёсткости (дискомфорт) |

### Нормативные требования (СП 52.13330)

| Помещение | Ē, лк | U₀ | UGR |
|---|---|---|---|
| Офис (работа за компьютером) | 400 | 0,6 | ≤ 19 |
| Офис (общее освещение) | 300 | 0,4 | ≤ 19 |
| Коридор | 100 | 0,4 | ≤ 25 |
| Склад | 100–200 | 0,4 | ≤ 25 |
| Производственный цех | 200–500 | 0,6 | ≤ 22 |
| Торговый зал | 400 | 0,6 | ≤ 22 |
| Учебный класс | 400 | 0,6 | ≤ 19 |

### Визуализация

- **Изолинии (изолюксы)** — контурная карта освещённости на рабочей плоскости
- **Цветовая карта** — тепловая карта (красный = много, синий = мало)
- **3D-рендер** — визуализация помещения с освещением
- **Диаграмма на стенах** — распределение освещённости по стенам

## 6. Формирование отчёта

### Состав отчёта для экспертизы

1. **Титульный лист** — проект, заказчик, проектировщик
2. **Описание проекта** — помещения, нормы, коэффициент запаса
3. **Список светильников** — тип, мощность, количество, координаты
4. **План размещения** — вид сверху с расположением светильников
5. **Результаты расчёта** — таблица (Ē, E_min, E_max, U₀, UGR)
6. **Изолинии** — распределение освещённости
7. **Удельная мощность** — Вт/м² (должна соответствовать СП)

### Экспорт

| Формат | Назначение |
|---|---|
| PDF | Отчёт для заказчика и экспертизы |
| DWG | План размещения в AutoCAD |
| CSV | Таблица результатов |

## 7. Продвинутые функции

### Расчёт аварийного освещения

DIALux Evo позволяет рассчитать аварийное освещение:
- Добавить аварийные светильники отдельным слоем
- Указать нормы (1 лк на путях эвакуации)
- Отключить рабочее освещение → рассчитать только аварийное

### Расчёт наружного освещения

- Создать наружную сцену
- Импортировать план территории (DWG)
- Задать расчётные поверхности (дороги, тротуары, площадки)
- Установить опоры с консольными светильниками

### Расчёт дорожного освещения

Специальный модуль для расчёта по EN 13201 / СП 52.13330:
- Задать параметры дороги (ширина, покрытие, класс)
- Установить опоры с шагом
- Получить яркость (L), равномерность (U₀, U_l)

### Tunable White и RGBW

- Создать несколько сценариев освещения
- Задать разную цветовую температуру для каждого сценария
- Рассчитать освещённость для каждого режима

## 8. Частые ошибки в DIALux

| Ошибка | Последствия | Решение |
|---|---|---|
| MF = 1,0 | Завышенный результат, не пройдёт экспертизу | Установить MF 0,67–0,80 |
| Неверные коэффициенты отражения | Завышенная/заниженная Ē | Задать реальные значения |
| Светильник без фотометрии | DIALux считает как точечный источник | Использовать .ies/.ldt |
| Рабочая плоскость на полу (h=0) | Завышенные результаты | Установить h = 0,8 м |
| Нет расчёта UGR | Замечание экспертизы | Включить расчёт UGR |
| Одно помещение на весь этаж | Некорректные отражения | Разбить на отдельные комнаты |

## 9. Быстрый чек-лист проекта

- [ ] Помещения построены с правильными размерами и высотами
- [ ] Коэффициенты отражения заданы (потолок / стены / пол)
- [ ] Коэффициент запаса (MF) задан по нормам
- [ ] Светильники с реальными фотометрическими файлами (.ies / .ldt)
- [ ] Высота рабочей плоскости = 0,8 м (или по нормам)
- [ ] Результаты: Ē, U₀, UGR — соответствуют нормам
- [ ] Удельная мощность ≤ допустимой по СП
- [ ] Сформирован PDF-отчёт со всеми разделами
- [ ] Экспортирован DWG с планом размещения

## Итоги

- DIALux Evo — бесплатный стандарт отрасли для расчёта освещения
- Всегда использовать реальные фотометрические файлы (.ies / .ldt)
- Коэффициент запаса (MF) — критически важен для корректного расчёта
- Результат расчёта — Ē, U₀, UGR — проверять по нормам СП 52.13330
- Отчёт в PDF — основной документ для экспертизы и заказчика
