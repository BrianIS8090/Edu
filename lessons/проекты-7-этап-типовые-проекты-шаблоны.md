# Типовые проекты как шаблоны — переиспользование наработок

Каждый новый заказ в 7Lamp на 60-80% повторяет предыдущие. Те же типы корпусов, те же драйверы, похожие крепления. Но инженеры раз за разом создают проекты с нуля — копируют файлы вручную, ищут чертежи в старых папках, заново собирают спецификации. Этот урок показывает, как выстроить систему шаблонов проектов: что именно можно переиспользовать между заказами, как организовать шаблон и как быстро адаптировать его под конкретного клиента.

---

## Зачем нужны шаблоны проектов

Представьте ситуацию: заказчик просит линейный светильник длиной 1200 мм для офиса. Инженер открывает пустую папку и начинает с нуля. Через три часа он понимает, что месяц назад делал почти такой же светильник — только длиной 900 мм. Модель была, чертежи были, спецификация была. Но найти их в хаосе папок не удалось.

**Шаблон проекта** — это заранее подготовленная структура, которая содержит всё типовое для определённого класса изделий: папки, базовые 3D-модели, чертежи-заготовки, стандартные спецификации и справочные документы. Шаблон не заменяет проектирование — он убирает рутину и позволяет сразу перейти к инженерной работе.

Выгоды измеримы:

* **Скорость** — запуск нового проекта занимает 15 минут вместо 2-3 часов
* **Качество** — шаблон уже содержит проверенные решения, повторяющиеся ошибки исключены
* **Единообразие** — все проекты команды выглядят одинаково, любой инженер может подхватить работу коллеги
* **Масштабирование** — новый сотрудник работает по шаблону с первого дня, без длительного погружения

## Типовые серии светильников: что можно переиспользовать

В 7Lamp основная часть заказов приходится на несколько серий изделий. Каждая серия имеет устойчивую конструктивную базу, которая меняется от заказа к заказу минимально.

### Линейные светильники (Line)

Самая массовая серия. Корпус — алюминиевый профиль, различия между заказами сводятся к длине, типу рассеивателя и мощности LED-модуля.

Что переиспользуется между заказами:

* 3D-модель профиля (параметрическая, меняется только длина)
* Узлы торцевых заглушек и подвесов
* Электросхема подключения LED-линейки к драйверу
* Спецификация крепежа и клемм
* Шаблон чертежа с основной надписью

### Даунлайты и трековые

Корпус — точёная или фрезерованная деталь. Между заказами меняется диаметр врезного отверстия, угол луча и цветовая температура LED.

Что переиспользуется:

* Параметрическая модель корпуса (меняется диаметр и глубина)
* Типовая оптическая сборка: LED-модуль + линза + радиатор
* Электросхема с драйвером и диммированием
* Чертёж установочного отверстия

### Архитектурные и фасадные

Влагозащищённые корпуса, IP65-IP67. Между заказами меняется форм-фактор и световое распределение.

Что переиспользуется:

* Узлы герметизации (сальники, уплотнители, заливка)
* Типовые решения по теплоотводу
* Спецификация влагозащитных компонентов
* Шаблон паспорта с указанием степени защиты

### Общая закономерность

Посмотрите на таблицу — она показывает, какой процент элементов типового проекта переходит в новый заказ без изменений или с минимальной правкой.

| Элемент проекта | Переиспользование | Что обычно меняется |
|-----------------|-------------------|---------------------|
| Структура папок | 100% | Ничего |
| Основная надпись чертежей | 90% | Название изделия, дата |
| Электросхема | 70-80% | Мощность драйвера, количество LED |
| Базовая 3D-модель | 60-80% | Габариты, отдельные узлы |
| Спецификация компонентов | 50-70% | Количество, отдельные позиции |
| Файлы для подрядчиков | 40-60% | Размеры деталей |

> Даже при 50% переиспользовании вы экономите половину времени на рутинных операциях. Остальное время уходит на инженерные решения — именно на то, за что платит заказчик.

## Структура шаблона проекта

Шаблон — это папка с фиксированной структурой. Каждая подпапка соответствует этапу работы над изделием. Ниже — эталонная структура для 7Lamp.

```
ШАБЛОН_[Серия]/
├── 01_ТЗ/
│   ├── бриф-заказчика.docx
│   └── чеклист-анализа-тз.md
├── 02_Модели/
│   ├── базовая-сборка.3dm
│   ├── узлы/
│   │   ├── крепление-подвес.3dm
│   │   ├── торцевая-заглушка.3dm
│   │   └── оптический-узел.3dm
│   └── README-параметры.md
├── 03_Чертежи/
│   ├── шаблон-сборочный.3dm
│   ├── шаблон-деталировка.3dm
│   └── основная-надпись.3dm
├── 04_Подрядчики/
│   ├── шаблон-задание-на-резку.dxf
│   ├── шаблон-задание-на-фрезеровку.step
│   └── требования-к-подрядчику.md
├── 05_Спецификация/
│   ├── типовая-спецификация.xlsx
│   └── справочник-компонентов.md
├── 06_Паспорт/
│   ├── шаблон-паспорта.docx
│   └── шаблон-инструкции-монтажа.docx
├── 07_Справочные/
│   ├── datasheet-led.pdf
│   ├── datasheet-драйвер.pdf
│   └── стандарты-и-нормы.md
└── ШАБЛОН-README.md
```

### Назначение каждой папки

**01_ТЗ** — шаблон брифа для заказчика и чеклист, по которому инженер анализирует входящее техническое задание. Бриф содержит типовые вопросы: тип помещения, требуемая освещённость, способ монтажа, бюджет.

**02_Модели** — базовая 3D-сборка в Moi3D/Rhino. Для линейных светильников это параметрическая модель профиля с типовыми узлами. Файл `README-параметры.md` описывает, какие размеры нужно изменить при адаптации.

**03_Чертежи** — шаблоны чертежей с заполненной основной надписью (кроме названия изделия и даты), рамками и типовыми видами. Инженер открывает шаблон и сразу работает с геометрией, а не тратит время на оформление.

**04_Подрядчики** — заготовки файлов для передачи на производство. DXF для лазерной резки, STEP для фрезеровки. Файл `требования-к-подрядчику.md` содержит допуски, покрытия, стандарты качества.

**05_Спецификация** — типовая спецификация с позициями, которые встречаются в каждом заказе данной серии. Справочник компонентов содержит артикулы, поставщиков и цены.

**06_Паспорт** — шаблон паспорта изделия и инструкции по монтажу. Структура и оформление готовы, осталось подставить конкретные параметры.

**07_Справочные** — даташиты основных компонентов серии, выдержки из стандартов. Инженеру не нужно каждый раз искать документацию — она лежит в проекте.

**ШАБЛОН-README.md** — инструкция по использованию шаблона: что менять, что оставить, порядок действий.

## Базовые 3D-модели: принципы создания

3D-модель в шаблоне должна быть не финальным изделием, а конструктором. Вот три принципа, которые делают модель пригодной для переиспользования.

### Параметричность

Ключевые размеры вынесены в отдельные параметры. Для линейного светильника это длина корпуса, количество LED-модулей и расстояние между подвесами. Изменение одного параметра автоматически перестраивает всю сборку.

### Модульность узлов

Каждый функциональный узел хранится отдельным файлом: торцевая заглушка, оптический блок, крепление подвеса, клеммная коробка. При сборке нового изделия инженер комбинирует готовые узлы, а не моделирует каждый с нуля.

### Документирование параметров

Файл `README-параметры.md` содержит таблицу изменяемых размеров с допустимыми диапазонами:

| Параметр | Диапазон | По умолчанию | Комментарий |
|----------|----------|--------------|-------------|
| Длина корпуса | 300-2400 мм | 1200 мм | Шаг 300 мм для стандартных профилей |
| Количество LED-модулей | 1-8 шт. | 4 шт. | Зависит от длины корпуса |
| Расстояние между подвесами | 400-1000 мм | 600 мм | Не более 1/3 длины корпуса |
| Тип рассеивателя | Опал / Призма / Микропризма | Опал | Влияет на КПД и UGR |

> Без документации параметров шаблон превращается в чёрный ящик. Новый сотрудник не поймёт, что можно менять, а что нельзя трогать.

## Типовые чертежи и спецификации

### Чертежи-заготовки

Шаблон чертежа содержит:

* Основную надпись с заполненными постоянными полями (организация, руководитель, стандарт оформления)
* Рамку по ГОСТ 2.104
* Типовую компоновку видов для данной серии (вид спереди, сбоку, разрез)
* Стандартные выноски и обозначения, характерные для серии

Инженер открывает шаблон, меняет название изделия в штампе, подставляет актуальную геометрию и дорабатывает размерные цепочки. Время на оформление сокращается с 2 часов до 20 минут.

### Типовые спецификации

Спецификация в шаблоне содержит позиции, которые присутствуют в каждом изделии серии. Для линейного светильника это:

* Алюминиевый профиль (артикул, поставщик)
* Рассеиватель (артикул, тип)
* Торцевые заглушки (2 шт.)
* Подвесы или клипсы
* LED-линейка (тип, параметры)
* Драйвер (серия, диапазон мощностей)
* Провод ШВВП 2x0.75 (типовая длина)
* Клеммы Wago (типовое количество)
* Винты, шайбы, заклёпки (типовой набор)

При адаптации инженер корректирует количество, выбирает конкретную модель драйвера под требуемую мощность и добавляет нестандартные позиции, если заказ отличается от типового.

## Как адаптировать шаблон под конкретный заказ

Адаптация шаблона — это не просто копирование папки. Это управляемый процесс с чётким порядком действий.

### Пошаговый процесс

1. **Скопируйте шаблон.** Создайте копию папки шаблона и переименуйте: `[Код клиента] Название изделия` — например, `[АРХ-052] Line 1500 офис Сколково`. Ни в коем случае не работайте в самом шаблоне.

2. **Заполните бриф.** Откройте `01_ТЗ/бриф-заказчика.docx` и внесите данные из ТЗ заказчика. Пройдите чеклист анализа — он покажет, все ли параметры определены.

3. **Адаптируйте модель.** Откройте базовую сборку в `02_Модели/`. Измените параметры по таблице из README. Замените узлы, если заказ требует нестандартного решения (например, поворотный подвес вместо жёсткого).

4. **Обновите чертежи.** Откройте шаблон чертежа, подставьте актуальную геометрию, заполните штамп: название изделия, дату, номер проекта. Проверьте размерные цепочки.

5. **Скорректируйте спецификацию.** Откройте типовую спецификацию, обновите количество позиций, замените драйвер на модель нужной мощности, добавьте нестандартные компоненты. Перенесите итоговую спецификацию в ERP.

6. **Подготовьте файлы для подрядчиков.** Экспортируйте обновлённую геометрию в DXF/STEP. Проверьте файл `требования-к-подрядчику.md` — возможно, нужно дополнить его специфическими условиями заказа.

7. **Оформите паспорт.** Заполните шаблон паспорта конкретными параметрами изделия: мощность, световой поток, цветовая температура, степень защиты, вес.

### Что нельзя менять в шаблоне

* Структуру папок — она стандартизирована для всей команды
* Файл `ШАБЛОН-README.md` — его можно только дополнить примечаниями к конкретному заказу
* Формат основной надписи чертежей — он соответствует ГОСТ 2.104

### Что нужно менять обязательно

* Название изделия во всех документах
* Габаритные размеры и параметры модели
* Количество и номенклатуру компонентов в спецификации
* Даты в основной надписи
* Параметры в паспорте изделия

## Управление библиотекой шаблонов

Шаблоны живут и развиваются. Один раз создать шаблон недостаточно — нужно поддерживать его в актуальном состоянии.

### Где хранить шаблоны

Шаблоны хранятся в общем сетевом хранилище в папке `_ШАБЛОНЫ/`. Доступ на чтение — у всех инженеров. Доступ на редактирование — только у ответственного за серию.

```
_ШАБЛОНЫ/
├── ШАБЛОН_Line/
├── ШАБЛОН_Downlight/
├── ШАБЛОН_Track/
├── ШАБЛОН_Facade/
└── ШАБЛОН_Custom/
```

Шаблон `Custom` — минимальная структура для нестандартных заказов, которые не вписываются ни в одну серию. Он содержит только структуру папок, шаблоны документов и пустую спецификацию.

### Обновление шаблонов

Обновляйте шаблон после каждого завершённого проекта, если в процессе работы появились улучшения:

* Новый узел, который может пригодиться в будущем
* Исправленная ошибка в типовом чертеже
* Обновлённый компонент в спецификации (например, драйвер снят с производства, замена на новую модель)
* Дополнительный пункт в чеклисте анализа ТЗ

> Правило: завершили проект — потратьте 15 минут на обновление шаблона. Эти 15 минут сэкономят часы в следующих проектах.

### Версионирование

Ведите лог изменений в файле `ШАБЛОН-README.md`:

```
## История изменений

2026-03-15 — Обновлён драйвер: Mean Well HLG-150H-24B заменён на HLG-150H-24AB
2026-02-20 — Добавлен узел поворотного подвеса (02_Модели/узлы/)
2026-01-10 — Шаблон создан на основе проекта [АРХ-041] Line 1200
```

## Частые ошибки при работе с шаблонами

| Ошибка | Последствие | Решение |
|--------|------------|---------|
| Работа прямо в папке шаблона | Шаблон испорчен для всей команды | Всегда копировать, никогда не редактировать оригинал |
| Шаблон не обновляется | Содержит устаревшие компоненты и ошибки | Обновлять после каждого проекта серии |
| Нет README с параметрами | Новый сотрудник не знает, что менять | Документировать каждый изменяемый параметр |
| Слишком детализированный шаблон | Адаптация занимает столько же, сколько создание с нуля | Включать только то, что повторяется в 70%+ заказов |
| Один шаблон на все серии | Не учитывает специфику конструкции | Отдельный шаблон для каждой серии |
| Забыли обновить название в штампе | Заказчик получает чертёж с чужим названием | Использовать чеклист адаптации |

## Чеклист создания нового шаблона

Используйте этот чеклист при формировании шаблона для новой серии изделий.

- [ ] Выбран завершённый проект-донор — самый чистый и полный из серии
- [ ] Создана стандартная структура папок (01_ТЗ ... 07_Справочные)
- [ ] 3D-модель очищена от заказо-специфичных деталей и параметризована
- [ ] Узлы вынесены в отдельные файлы
- [ ] Создан README-параметры.md с таблицей изменяемых размеров
- [ ] Чертежи очищены от конкретных размеров, основная надпись подготовлена
- [ ] Типовая спецификация содержит только повторяющиеся позиции
- [ ] Шаблон паспорта очищен от конкретных параметров
- [ ] Файлы для подрядчиков обобщены
- [ ] ШАБЛОН-README.md написан с инструкцией по адаптации
- [ ] Шаблон размещён в общем хранилище с корректными правами доступа
- [ ] Команда уведомлена о новом шаблоне
