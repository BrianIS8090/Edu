# Фотометрия и IES/LDT-файлы

Фотометрические данные — язык общения между производителем светильника и проектировщиком. Без корректных IES/LDT-файлов невозможно сделать расчёт в DIALux, выиграть тендер или доказать соответствие нормам. Этот урок — от физики до практики работы с файлами.

---

## 1. Основы фотометрии

### Фотометрическое тело

Представьте светильник в центре сферы. Из каждой точки светильника свет распространяется с разной интенсивностью в разных направлениях. Если из центра светильника отложить векторы, длина которых пропорциональна силе света в этом направлении, получится **фотометрическое тело** — 3D-форма, описывающая распределение света.

### Ключевые величины

| Величина | Обозначение | Единица | Описание |
|---|---|---|---|
| Световой поток | Φ | лм (люмен) | Общее количество видимого света |
| Сила света | I | кд (кандела) | Плотность потока в направлении |
| Освещённость | E | лк (люкс) | Поток на единицу площади (лк = лм/м²) |
| Яркость | L | кд/м² | Сила света с единицы видимой площади |

### Связь между величинами

```
Освещённость:   E = I / d²  (закон обратных квадратов)
Световой поток:  Φ = ∫∫ I(θ,φ) · sin(θ) dθ dφ  (интеграл по сфере)
```

**Практический смысл:** Удвоение расстояния → освещённость падает в 4 раза.

## 2. Кривая силы света (КСС)

### Что это

КСС — это сечение фотометрического тела плоскостью, проходящей через ось светильника. Показывает, как сила света зависит от угла.

### Системы координат

**C-γ (C-gamma)** — стандарт для большинства светильников:
- **C** — азимутальный угол (плоскость сечения), 0–360°
- **γ (gamma)** — полярный угол от надира (вниз = 0°, горизонт = 90°, вверх = 180°)

Основные плоскости:
- **C0–C180** — поперечная плоскость
- **C90–C270** — продольная плоскость

**B-β (B-beta)** — для прожекторов:
- **B** — угол поворота светильника
- **β** — угол от оси

### Типовые формы КСС

| Тип | Форма | Применение |
|---|---|---|
| Косинусная (D) | Широкое распределение | Общее освещение, панели |
| Глубокая (Г) | Узкий конус вниз | Даунлайты, высокие потолки |
| Полуширокая (Л) | Вытянутая в стороны | Уличные светильники |
| Широкая (Ш) | Очень широкое распределение | Светильники для низких потолков |
| Концентрированная (К) | Очень узкий луч | Прожекторы, споты |
| Равномерная (М) | Сферическое распределение | Декоративные подвесы |
| Асимметричная | Свет направлен в одну сторону | Настенные, витринные |

### Как читать полярную диаграмму

```
         90°
          |
    120°  |  60°
     \    |    /
      \   |   /
       \  |  /    ← линия = сила света (кд) при данном угле
        \ | /
         \|/
  180° ───●─── 0° (вниз / надир)
```

- Чем дальше линия от центра — тем сильнее свет в этом направлении
- Масштаб подписан (кд/1000 лм или кд)
- Разные линии = разные плоскости (C0, C90 и т.д.)

## 3. Гониофотометр — как измеряют

### Типы гониофотометров

| Тип | Принцип | Применение |
|---|---|---|
| Тип A (зеркальный) | Вращающееся зеркало, светильник неподвижен | Большие светильники (не нагреваются) |
| Тип B | Светильник вращается вокруг двух осей | Прожекторы, малые светильники |
| Тип C | Светильник вращается вокруг вертикальной оси, датчик движется по дуге | Наиболее распространённый |

### Условия измерения

| Параметр | Требование |
|---|---|
| Расстояние до фотоприёмника | ≥ 5× максимальный габарит светильника (обычно 10–25 м) |
| Температура в лаборатории | 25 ± 1 °C |
| Стабилизация светильника | ≥ 30 мин (до теплового равновесия) |
| Паразитная засветка | Тёмная комната с чёрными стенами |
| Шаг измерения | Обычно 2,5° или 5° (для точных — 1°) |
| Результат | Файл .ies или .ldt |

### Интегрирующая сфера (дополнительно)

Для измерения общего светового потока без углового распределения:

- Диаметр сферы ≥ 6× размер светильника
- Покрытие — сульфат бария (BaSO₄) или Spectralon
- Результат — световой поток в люменах

## 4. Формат IES (IESNA LM-63)

### Структура файла

```
IESNA:LM-63-2002
[TEST] Laboratory Test Report #12345
[MANUFAC] 7Lamp
[LUMCAT] LINE-1200-40-4000K
[LUMINAIRE] Linear LED luminaire 1200mm 40W 4000K
[LAMP] LED module
[OTHER] CRI Ra>90
TILT=NONE
1 4000 1.0 37 1 1 2 0.04 0.06 1.2
1.0 1.0 1.0
0.0 2.5 5.0 7.5 10.0 ... 175.0 177.5 180.0
0.0 90.0 180.0 270.0
2143 2138 2125 2103 ... 0 0 0
2143 2135 2118 2090 ... 0 0 0
...
```

### Расшифровка ключевых строк

| Строка | Значение |
|---|---|
| IESNA:LM-63-2002 | Версия формата |
| [TEST] | Номер протокола испытаний |
| [MANUFAC] | Производитель |
| [LUMCAT] | Каталожный номер |
| TILT=NONE | Нет данных о наклоне (типично для внутренних) |
| Строка параметров | Кол-во ламп, поток, множитель, кол-во углов γ и C, тип, размеры |
| Углы γ | Список вертикальных углов (0, 2.5, 5, ... 180) |
| Углы C | Список горизонтальных плоскостей (0, 90, 180, 270) |
| Данные I(γ) | Сила света в кд для каждой комбинации C и γ |

### Типы симметрии в IES

| Количество плоскостей C | Симметрия |
|---|---|
| 1 плоскость (C=0) | Полная осевая симметрия (круглые светильники) |
| 2 плоскости (C=0, C=90) | Симметрия по двум осям (линейные) |
| 4 плоскости (C=0, C=90, C=180, C=270) | Симметрия по одной оси |
| Все плоскости (C=0...360) | Нет симметрии (асимметричные) |

## 5. Формат EULUMDAT (LDT)

### Структура файла

Текстовый файл, каждая строка — отдельный параметр:

```
Строка 1:  Идентификатор компании
Строка 2:  Тип данных (1=точечный, 2=линейный, 3=площадной)
Строка 3:  Симметрия (0=нет, 1=C0, 2=C0-C180, 3=C0-C270, 4=C0-C360)
Строка 4:  Количество плоскостей C (Mc)
Строка 5:  Шаг между плоскостями C (Dc), градусы
Строка 6:  Количество углов γ в каждой плоскости (Ng)
Строка 7:  Шаг между углами γ (Dg), градусы
Строка 8:  Номер отчёта об измерениях
Строка 9:  Наименование светильника
Строка 10: Каталожный номер
...
Строка 27: Световой поток лампы (лм)
Строка 28: Цветовая температура (К)
...
```

Далее — массив значений силы света.

### LDT vs IES — сравнение

| Параметр | IES (LM-63) | LDT (EULUMDAT) |
|---|---|---|
| Происхождение | США (IESNA) | Европа (Германия) |
| Формат | Ключевые слова + данные | Фиксированная структура строк |
| Поддержка в DIALux | Да | Да |
| Распространённость в РФ | Средняя | Высокая (европейский стандарт) |
| Поддержка в Relux | Да | Да |

## 6. Просмотр и редактирование файлов

### Программы для просмотра

| Программа | Бесплатная | Возможности |
|---|---|---|
| Photometric Viewer (Lighting Analysts) | Да | Просмотр IES/LDT, 3D-визуализация |
| IES Viewer | Да | Простой просмотр IES |
| DIALux Evo | Да | Полноценный расчёт |
| Relux Desktop | Да | Полноценный расчёт |
| Photopia | Нет | Профессиональное проектирование оптики |

### Что проверять в файле

1. **Световой поток** — соответствует ли заявленному в даташите
2. **Форма КСС** — адекватна ли для данного типа светильника
3. **Симметрия** — правильно ли задана
4. **Количество углов** — достаточно ли для точного расчёта (шаг ≤ 5°)
5. **Размеры светящей части** — влияют на расчёт UGR
6. **Данные TILT** — для наклоняемых светильников

## 7. Создание и корректировка IES/LDT

### Когда нужно создавать файл

- Новый светильник — после фотометрических измерений в лаборатории
- Модификация существующего — изменение оптики, LED-модуля, мощности
- Нет данных от производителя компонентов

### Способы получения

| Способ | Точность | Стоимость |
|---|---|---|
| Измерение на гониофотометре | Эталонная | Высокая (лаборатория) |
| Моделирование в Photopia / TracePro | Высокая (при правильной модели) | Средняя (лицензия ПО) |
| Масштабирование существующего файла | Приблизительная | Бесплатно |
| Использование типовых КСС от производителя LED | Ориентировочная | Бесплатно |

### Масштабирование файла

Если изменилась только мощность (LED тот же, оптика та же):

```
Φ_новый = Φ_старый × (P_новый / P_старый)

Способ: умножить все значения силы света на коэффициент Φ_новый / Φ_старый
```

**Внимание:** Это корректно только при линейном изменении потока от мощности. При существенном изменении мощности — переизмерять.

## 8. Использование в проектировании

### В DIALux Evo

1. Импортировать .ies / .ldt (Файл → Импорт светильника)
2. Проверить в превью: КСС, световой поток, размеры
3. Разместить на плане
4. Рассчитать → получить Ē, U₀, UGR

### В тендерной документации

Фотометрические файлы часто требуются в техническом задании:

- Файл .ies / .ldt — обязательное приложение
- Протокол фотометрических испытаний — подтверждение
- Расчёт DIALux с использованием предоставленных файлов

### КПД светильника (LOR / Efficiency)

```
КПД = Φ_светильника / Φ_источника × 100%
```

| Тип | Типичный КПД |
|---|---|
| Даунлайт с отражателем | 70–85% |
| Панель с рассеивателем (опал) | 80–90% |
| Линейный с призмой | 85–92% |
| Прожектор с линзой | 85–95% |
| Без оптики (открытый) | 95–100% |

## 9. Типичные ошибки

| Ошибка | Последствия | Решение |
|---|---|---|
| Использование чужого файла «похожего» светильника | Некорректный расчёт, несоответствие нормам | Измерить или смоделировать свой |
| Файл без указания размеров светящей части | UGR рассчитывается неверно | Указать реальные размеры |
| Данные при 25°C, а светильник работает при 40°C | Реальный поток ниже на 10–15% | Учесть температурную деградацию |
| Устаревший файл (старая модификация) | Несоответствие реальному изделию | Актуализировать при каждом изменении |
| Слишком грубый шаг (15–30°) | Неточная форма КСС, ошибки в расчётах | Шаг ≤ 5° для коммерческих файлов |
| Нет данных о мультипликаторе потока | DIALux использует значение из файла как абсолютное | Проверить соответствие даташиту |

## Итоги

- **Фотометрическое тело** — 3D-распределение силы света; **КСС** — его сечение
- **IES** (американский) и **LDT** (европейский) — два основных формата файлов
- Измерение на **гониофотометре** — единственный способ получить точные данные
- Всегда проверять: поток, форму КСС, размеры, симметрию
- Файлы нужны для расчёта в **DIALux**, для **тендеров** и для **сертификации**
- При изменении оптики или LED — файл нужно **переизмерить**
