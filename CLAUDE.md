# EduPlatform — Инструкция

Обучающая платформа для инженеров компании Group 7Lamp. Единая точка входа для всех обучающих материалов: Markdown-уроки, интерактивные HTML-модули, видео, PDF, презентации.

## Архитектура

SPA на чистом HTML/CSS/JS без фреймворков. Контент добавляется через git push — платформа подхватывает автоматически.

```
Edu/
├── index.html                  # Единая точка входа
├── app.js                      # Клиентская логика
├── assets/                     # Материалы уроков (видео, PDF, картинки)
│   └── <название-урока>/
├── lessons/                    # Markdown-уроки с frontmatter
│   └── <название>.md
├── modules/                    # Интерактивные HTML-модули
│   └── <название>/
│       ├── index.html
│       └── meta.md
├── build/
│   └── catalog.json            # Автогенерируемый индекс (НЕ редактировать вручную)
├── scripts/
│   └── build-catalog.sh        # Скрипт сборки catalog.json
└── .github/
    └── workflows/
        └── deploy-pi.yml       # CI/CD: сборка + деплой на Pi5
```

## Как это работает

1. Автор создаёт Markdown-файл или HTML-модуль
2. Пушит в GitHub (`git push`)
3. GitHub Actions запускает `scripts/build-catalog.sh` — сканирует `lessons/` и `modules/`, парсит frontmatter, генерирует `build/catalog.json`
4. Деплоит на Raspberry Pi 5 (Nginx)
5. Пользователь открывает сайт → `app.js` загружает `catalog.json` → строит sidebar и список материалов
6. Клик на урок → fetch Markdown → рендер через marked.js
7. Клик на модуль → загрузка HTML в iframe

## Именование файлов

### Уроки (`lessons/*.md`)

**Формат:** `<префикс-категории>-<название-через-дефисы>.md`

| Категория | Префикс |
|-----------|---------|
| Светотехника | `светотехника` |
| Проектирование освещения | `проектирование` |
| Проекты | `проекты` |
| Электроника | `электроника` |
| Продажи | `продажи` |
| Маркетинг | `маркетинг` |
| ИИ и агенты | `агенты` |
| Онбординг | `обучение` |

**Правила:**
- Префикс — строчные буквы, кириллица
- Разделитель между префиксом и названием — дефис `-`
- Пробелы в названии заменяются дефисами
- Без подчёркиваний, без заглавных букв
- Примеры:
  - `светотехника-нормы-освещённости.md`
  - `агенты-генерация-изображений.md`
  - `обучение-установка-openclaw.md`
  - `проекты-3d-моделирование-cad-системы.md`

### Модули (`modules/<название>/`)

**Формат:** kebab-case на латинице

- Всё строчными буквами
- Слова разделяются дефисами
- Без пробелов, подчёркиваний, кириллицы
- Примеры: `color-science/`, `heatsink-calc/`, `pwm-dimming/`

### Ассеты (`assets/<название>/`)

**Формат:** совпадает с названием урока **без префикса категории**

- Примеры:
  - Урок `светотехника-нормы-освещённости.md` → `assets/нормы-освещённости/`
  - Урок `инженеры-работа-с-7lamp-erp-v8.md` → `assets/работа-с-7lamp-erp-v8/`
- Дефисы вместо пробелов
- Кириллица допускается

## Как добавить Markdown-урок

### 1. Создать файл в `lessons/`

Имя файла по конвенции: `lessons/<префикс-категории>-<название>.md`, например `lessons/светотехника-основы-оптики.md`.

### 2. Добавить frontmatter в начало файла

```markdown
---
title: Основы светотехники
category: Светотехника
subcategory: Основы фотометрии
tags: [свет, люмены, люксы]
---

# Содержимое урока

Обычный Markdown (GFM). Заголовки, списки, ссылки, картинки, блоки кода — всё работает.
```

**КРИТИЧЕСКИ ВАЖНО — формат frontmatter:**

* Блок ОБЯЗАТЕЛЬНО обёрнут в `---` сверху и снизу

* Между полями НЕ должно быть пустых строк

* Frontmatter идёт ПЕРВЫМ в файле, до любого контента

**Обязательные поля frontmatter:**

* `title` — название урока (отображается на платформе)

* `category` — категория для sidebar (например: Светотехника, Проектирование освещения, Электроника)

* `subcategory` — подкатегория внутри категории (например: DIALux, Нормы освещения, Драйверы)

* `tags` — теги для фильтрации, формат: `[тег1, тег2, тег3]`

### 3. Если есть вложения (видео, PDF, картинки)

Создать папку в `assets/` с тем же именем и положить туда файлы:

```
assets/
  основы-светотехники/
    видео.mp4
    схема.pdf
    диаграмма.png
```

В Markdown ссылаться так:

```markdown
![Диаграмма](assets/основы-светотехники/диаграмма.png)
[Скачать PDF](assets/основы-светотехники/схема.pdf)
```

### 4. Запушить

```bash
git add lessons/основы-светотехники.md assets/основы-светотехники/
git commit -m "Добавить урок: Основы светотехники"
git push
```

Урок автоматически появится на сайте.

## Как добавить интерактивный HTML-модуль

### 1. Создать папку в `modules/`

```
modules/
  новый-калькулятор/
    index.html    # Сам модуль (полноценная HTML-страница)
    meta.md       # Метаданные + описание
```

### 2. Создать meta.md

```markdown
---
title: Новый калькулятор
category: Светотехника
tags: [расчёт, LED]
---

Описание модуля в одну-две строки.
```

### 3. Запушить

```bash
git add modules/новый-калькулятор/
git commit -m "Добавить модуль: Новый калькулятор"
git push
```

## Локальная проверка (без git push)

**PowerShell (Windows):**

```powershell
# 1. Пересобрать каталог
.\scripts\build-catalog.ps1

# 2. Запустить локальный сервер
npx serve . -l 8080
```

**Bash (Linux/macOS/Git Bash):**

```bash
# 1. Пересобрать каталог
bash scripts/build-catalog.sh > build/catalog.json

# 2. Запустить локальный сервер
npx serve . -l 8080
```

Открыть <http://localhost:8080> — платформа работает локально.

После каждого добавления/изменения файлов нужно пересобрать каталог (шаг 1) и обновить страницу в браузере.

## Ссылки на файлы в Markdown

**Картинки (отображаются inline):**

```markdown
![Описание](assets/имя-папки/файл.png)
```

**Ссылки на скачивание:**

```markdown
[Скачать PDF](assets/имя-папки/файл.pdf)
```

**ВАЖНО:** Имена папок и файлов — через дефис или подчёркивание, без пробелов. Слэши только прямые `/`, не обратные `\`.

## Технологический стек

* **HTML/CSS/JS** — без фреймворков

* **marked.js** — рендеринг Markdown на клиенте (CDN)

* **Lucide** — иконки (CDN, <https://lucide.dev/icons/>)

* **Nginx** — веб-сервер на Raspberry Pi 5

* **GitHub Actions** — CI/CD (self-hosted runner на Pi5)

* **bash** — скрипт сборки catalog.json

## Правила для агентов

* Иконки — только из Lucide (<https://lucide.dev/icons/>), никаких emoji

* Комментарии в коде — на русском языке

* Отступ — 2 пробела

* `catalog.json` генерируется автоматически — НЕ редактировать вручную

* При добавлении нового урока/модуля всегда добавлять frontmatter

* Светлая тема, акцентный цвет: #4f46e5 (индиго)

* Шрифт: системный (-apple-system, Segoe UI)

* Шрифт кода: JetBrains Mono

## CI/CD Pipeline

```
git push → GitHub Actions → git pull на Pi5 → build-catalog.sh → rsync в /var/www/edu/
```

Сервер: Raspberry Pi 5, Nginx, путь: `/var/www/edu/`
